# src-tauri

## Purpose

Rust backend for the GlimmerX Tauri application. Handles all database operations, business logic, and exposes commands to the frontend via Tauri's IPC mechanism.

## Key Files

| File | Description |
|------|-------------|
| `Cargo.toml` | Rust dependencies including rusqlite (with bundled-sqlcipher), serde, chrono, uuid, thiserror |
| `tauri.conf.json` | Tauri app configuration: productName "GlimmerX", window 1200x800, devUrl localhost:11422 |
| `build.rs` | Tauri build script (auto-generated by scaffold) |
| `.gitignore` | Excludes target/, gen/, and other build artifacts |

## For AI Agents

### Working In This Directory

- Run `cargo clippy -- -D warnings` before committing — warnings block the commit
- Run `cargo fmt` before committing — formatting must match
- All commands exposed to frontend must return `Result<T, String>`
- Amounts stored as `i64` (cents) — never `f64` for money
- Double-entry rule: `sum(postings.amount) == 0` must be enforced on every transaction
- Database uses SQLCipher (encrypted SQLite) — password required for all operations
- **Time handling: use `utils::time` module** — All timestamps and dates must use local timezone via `chrono::Local`. NEVER use `chrono::Utc::now()`. Use `now_rfc3339()` for timestamps, `today_date()` for business dates.

### Testing Requirements

- `cargo test` — runs unit tests (138 tests as of 2026-04-21)
- Coverage requirement: >= 95% overall, 100% for critical modules (transaction validation, amount calculation, database encryption)
- Tests should use in-memory SQLite instances for isolation

### Common Patterns

- Tauri commands annotated with `#[tauri::command]`
- Models derive `Serialize` and `Deserialize` via serde
- Error handling via `thiserror` derive macros
- UUID v4 for all primary keys
- ISO 8601 strings for dates in the database (RFC3339 with local timezone offset for timestamps)
- WAL mode for SQLite concurrency
- Dashboard data computed via single aggregated queries where possible

### Module Structure
